## 第一章 开发环境分析

### 1.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | 纯 HTML5 + CSS + ES6 JavaScript | **无框架**，无 Webpack/Vite/React/Vue |
| 样式 | 原生 CSS（CSS 变量驱动） | 两份文件：`style.css`（全局）+ `home-ui-enhance.css`（首页动效） |
| 地图 | 百度地图 JS API GL v1.0 | CDN 动态注入，WebGL 渲染 |
| AI 接口 | 火山方舟 API v3 | 三个端点：chat/completions + bots + responses |
| 数据维护 | Python 3 脚本 | 16 个 `.py` 文件，用于数据抓取/翻译/打包 |
| 系统脚本 | PowerShell 5 / Batch | CSV 同步、本地服务器启动 |

### 1.2 运行环境要求

| 组件 | 最低版本 | 用途 | 检测方法 |
|------|----------|------|----------|
| Python | **3.8+** | 数据更新、翻译、打包、本地服务器 | `python --version` |
| pip（requests） | — | API 调用，数据抓取 | `python -c "import requests"` |
| pip（Pillow） | —（可选） | 图片处理（背景裁切/缩放） | `python -c "import PIL"` |
| pip（rembg） | —（可选） | 吉祥物抠图（仅生成资源时） | `python -c "import rembg"` |

| npm / pnpm | **不需要** | 无 package.json | — |
| 浏览器 | Chrome/Edge 最新版 | 访问页面 | — |

> 根据项目结构推测：**无 Node.js、无 Vite、无 Vue CLI、无 React 脚手架**。项目是纯静态页面，所有 JS 通过 `<script>` 标签顺序加载，通过 `window.XXX` 全局对象共享状态。

## 第四章 环境搭建指南

### 4.1 第一步：安装 Python

1. 打开 [python.org](https://www.python.org/downloads/)
2. 下载 Python 3.8 或更高版本（推荐 3.11）
3. 安装时**勾选**「Add Python to PATH」
4. 安装完成后验证：

```powershell
python --version
# 应输出：Python 3.11.x（或更高）
```

### 4.2 第二步：安装核心 Python 依赖

打开 PowerShell（以管理员身份），执行：

```powershell
pip install requests
```

验证安装：

```powershell
python -c "import requests; print('requests 安装成功，版本：' + requests.__version__)"
```

> 如果安装失败（网络问题），可用国内镜像：
> ```powershell
> pip install requests -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

### 4.3 第三步：配置 API 密钥

这是**最重要的一步**，缺失会导致首页侨情动态空白、地图不显示、侨壮壮无法回复。

1. 打开 `date/volc-ark-apis.json`
2. 替换以下 4 个关键字段（详细说明见第七章）：

| 字段 | 值来源 |
|------|--------|
| `dataApiKey` | 火山方舟 → API Key 管理 |
| `qiaozhuangApiKey` | 同上（可以与 dataApiKey 相同） |
| `baiduMapAk` | 百度地图开放平台 → 浏览器端应用 AK |
| `ARK_QIAOZHUANG_DEFAULTS.model` | 豆包智能体 → Bot ID（`bot-` 开头） |

3. 保存文件

> 详细获取步骤见 `date/API修改指南.md`

### 4.4 第四步：运行环境检查脚本

双击项目根目录的 `setup-env-check.py`，确认所有环境检查通过：

```
✅ Python 已安装
✅ requests 包可用
✅ 配置文件存在
✅ 百度地图 AK 已配置
✅ 火山方舟 API Key 已配置
⚠  ffmpeg 未安装（不影响日常使用）
⚠  Pillow 未安装（不影响日常使用）
```

## 第五章 项目启动流程

### 5.1 首次启动

**方式一：双击启动（推荐）**

直接双击项目根目录的 `打开网页.py`：
- 自动在 `127.0.0.1:8765`（或附近空闲端口）启动静态服务器
- 自动用默认浏览器打开 `index.html`
- 按 `Ctrl+C` 关闭

**方式二：命令行启动**

```powershell
cd d:\玉小侨demo
python 打开网页.py
```

**方式三：PowerShell 脚本**

```powershell
powershell -ExecutionPolicy Bypass -File scripts/start-local-server.ps1
```

### 5.2 浏览器直接打开（不推荐）

可以双击 `index.html` 直接用浏览器打开，但：
- ❌ CSV/JSON 数据因跨域限制可能加载失败
- ❌ 部分功能不可用
- ✅ 适合快速预览页面样式

### 5.3 日常启动（每日维护）

每天只需：

```powershell
cd d:\玉小侨demo
python 打开网页.py
```

无需重新安装任何依赖。

### 5.4 数据更新流程

```powershell
cd d:\玉小侨demo
python 一键更新数据部分.py
```

此脚本自动执行 5 步流水线：

| 步骤 | 脚本 | 作用 | 失败影响 |
|------|------|------|----------|
| 1 | `ensure_deps.py` | 检查依赖 | ❌ 阻断后续 |
| 2 | `sync-data.ps1` | CSV → datasets.data.js | ❌ 阻断后续 |
| 3 | `daily_data_update.py` | 产业/文化数据日更（24h 节流） | ⚠️ 不影响 |
| 4 | `fetch_qiaowu_to_date.py` | 侨情动态抓取 | ⚠️ 不影响 |
| 5 | `post_update_translate_data.py` | 离线多语翻译 | 默认跳过 |

### 5.5 单独运行某个脚本

```powershell
# 只更新侨情数据
cd d:\玉小侨demo
python scripts/fetch_qiaowu_to_date.py

# 只更新产业/文化数据
python scripts/daily_data_update.py

# 开启翻译（耗时长、计费）
$env:YXQ_POST_I18N_ENABLE = "1"
python scripts/post_update_translate_data.py
```

### 5.6 CSV 数据修改后重新生成

修改任何 CSV 后，运行：

```powershell
cd d:\玉小侨demo
powershell -ExecutionPolicy Bypass -File scripts/sync-data.ps1
```

这会重新生成 `date/datasets.data.js`（前端加载的数据源）。

### 5.7 前端构建

**本项目无需构建**。修改 HTML/CSS/JS 后直接刷新浏览器即可生效。
不需 `npm run build`、`vite build` 等命令。

## 第六章 配置项说明

### 6.1 配置文件清单

| 文件 | 类型 | 必须修改 | 说明 |
|------|------|----------|------|
| `date/volc-ark-apis.json` | API 密钥 | ✅ 是 | 核心配置（详见第七章） |
| `scripts/ark_defaults.py` | 默认值 | ❌ 否 | 火山方舟回退默认值（不含密钥） |
| `js/platform-data-api-config.js` | 外部 API 占位 | ❌ 否 | 为空，为将来对接后端预留 |

### 6.2 `volc-ark-apis.json` 字段详解

```jsonc
{
  // ======== API 密钥（必须修改）========
  "dataApiKey": "xxx",              // 数据生成 + 内容获取
  "translationApiKey": "",          // 翻译（空=复用 dataApiKey）
  "qiaozhuangApiKey": "xxx",        // 侨壮壮对话 Bot
  "baiduMapAk": "xxx",              // 百度地图（浏览器端）
  "baiduMapStyleId": "xxx",         // 个性化地图样式码

  // ======== 模型配置（一般不用改）========
  "ARK_DATA_CHAT_DEFAULTS": {
    "endpoint": "https://ark.cn-beijing.volces.com/api/v3/chat/completions",
    "model": "ep-xxxxxxxx-xxxxx"    // 数据生成推理接入点
  },
  "ARK_QIAOZHUANG_DEFAULTS": {
    "endpoint": "https://ark.cn-beijing.volces.com/api/v3/bots/chat/completions",
    "model": "bot-xxxxxxxx-xxxxx"   // 豆包智能体 ID
  },
  "ARK_TRANSLATION_RESPONSES_DEFAULTS": {
    "endpoint": "https://ark.cn-beijing.volces.com/api/v3/responses",
    "model": "ep-xxxxxxxx-xxxxx"    // 翻译推理接入点
  },

  // ======== 功能开关 ========
  "TRANSLATION_ARK":     { "enabled": true },
  "DATA_ACQUISITION_ARK": { "enabled": true, "temperature": 0.35, "max_tokens": 12288 },
  "ASSISTANT_ARK":       { "use_stream": true }
}
```

### 6.3 环境变量（Python 脚本）

| 变量 | 作用 | 默认值 |
|------|------|--------|
| `ARK_API_KEY` | 覆盖 `dataApiKey`（调试用） | 无 |
| `YXQ_POST_I18N_ENABLE` | 设为 `1` 开启更新后自动翻译 | 关闭 |
| `PAGER` | 无头模式下避免分页器挂起 | `cat` |

## 第七章 API 替换指南

### 7.1 全部外部服务一览

| 服务 | 用途 | 配置位置 | 账号绑定 |
|------|------|----------|----------|
| **火山方舟（数据生成）** | 产业/文化内容 AI 生成 | `dataApiKey` + `ARK_DATA_CHAT_DEFAULTS` | 火山方舟账号 |
| **火山方舟（翻译）** | 多语批量翻译 | `translationApiKey` + `ARK_TRANSLATION_RESPONSES_DEFAULTS` | 同上 |
| **火山方舟（侨壮壮）** | 侨壮壮智能体对话 | `qiaozhuangApiKey` + `ARK_QIAOZHUANG_DEFAULTS` | 同上 + 豆包智能体 |
| **百度地图 JS API** | 地图渲染、城市搜索 | `baiduMapAk` | 百度地图开放平台 |
| **百度地图个性化样式** | 深色主题 | `baiduMapStyleId` | 百度地图控制台 |

### 7.2 从开发者个人账号迁移到新账号

按以下顺序操作（每一步约 10 分钟）：

#### 步骤 1：火山方舟 API Key

1. 用新账号登录 [console.volcengine.com/ark](https://console.volcengine.com/ark)
2. 左侧 → **API Key 管理** → 创建新 Key
3. 复制 Key，替换 `volc-ark-apis.json` 中的：
   - `dataApiKey`
   - `qiaozhuangApiKey`（可与 dataApiKey 相同）

#### 步骤 2：豆包智能体（侨壮壮）

1. 登录 [豆包智能体平台](https://www.doubao.com/agent) 或火山方舟「智能体工作室」
2. 创建智能体：
   - **名称**：侨壮壮
   - **提示词**：

   ```
   你是"侨壮壮"，玉林侨务相关的数字助手，性格大方活泼，带着亲切热忱的气质，容易让人产生亲近感。

   核心是围绕玉林侨务相关事务提供帮助，包括解答侨胞关心的各类问题、分享真实的侨乡动态、衔接侨商合作等。

   所有回应必须基于客观事实，不编造信息，对于不确定的内容，会如实说明"这个信息我不太确定，建议咨询相关部门"。

   生成的回答里自动去掉ai痕迹，让对话显得自然，任何问题只做最多三百字的简单回答。

   面对提问时，相关内容要清晰回应，尽量给出实用、准确的信息；即使遇到不相关的话题，也先简单回应，再自然引导到侨务、地方发展等相关方向。

   用户可能用不同语言提问，需要回答相应的语言。

   说话风格轻松自然，不刻意使用特定称谓，以平和的语气沟通，让接触到的人感受到友好与热忱，同时聚焦于服务侨胞、促进交流的核心目标。
   ```

   - **欢迎语**：`你好呀！我是侨壮壮，玉林侨务相关的数字助手。有什么想了解的侨乡信息、侨商合作、政策解读，都可以问我～`
3. 发布智能体，复制 **Bot ID**（`bot-xxxx` 开头）
4. 替换 `volc-ark-apis.json` 中 `ARK_QIAOZHUANG_DEFAULTS.model`

#### 步骤 3：百度地图 AK

1. 用新账号登录 [lbsyun.baidu.com](https://lbsyun.baidu.com/)
2. 创建**浏览器端**应用
3. Referer 白名单填 `*`（测试）或生产域名
4. 复制 AK，替换 `volc-ark-apis.json` 中 `baiduMapAk`

#### 步骤 4：百度地图个性化样式

1. 百度地图控制台 → 个性化地图 → 样式管理
2. 创建/编辑深色主题样式，发布后复制样式 ID
3. 替换 `volc-ark-apis.json` 中 `baiduMapStyleId`

## 第九章 常见问题排查

### 9.1 Python 版本问题

| 现象 | 原因 | 解决 |
|------|------|------|
| `python` 命令不识别 | Python 未安装或未加 PATH | 重新安装 Python，勾选「Add Python to PATH」 |
| `pip` 命令不识别 | pip 未安装 | `python -m ensurepip --upgrade` |
| 脚本报 UnicodeError | Python 版本过低 | 升级至 Python 3.8+ |

### 9.2 依赖安装失败

| 现象 | 原因 | 解决 |
|------|------|------|
| `pip install requests` 超时 | 网络问题（PyPI 在国外） | 用清华镜像：`pip install requests -i https://pypi.tuna.tsinghua.edu.cn/simple` |
| 权限拒绝 | 缺少管理员权限 | 以管理员身份运行 PowerShell |
| `Pillow` 安装失败 | 缺少编译环境 | `pip install Pillow` 不需要编译；若失败用 `pip install --only-binary=:all: Pillow` |

### 9.3 API 失效

| 现象 | 原因 | 解决 |
|------|------|------|
| 侨情动态空白 | `dataApiKey` 无效或过期 | 更新 `date/volc-ark-apis.json` 中的 API Key |
| 侨壮壮回复空白 | `qiaozhuangApiKey` 或 Bot ID 不匹配 | 检查 Key 和 Bot 是否属于同一账号 |
| 地图显示"无权访问" | `baiduMapAk` 无效或 Referer 白名单不对 | 更新 AK → 后台设白名单 `*` |

### 9.4 CSV / 数据缺失

| 现象 | 原因 | 解决 |
|------|------|------|
| 页面内容加载不出来 | CSV 被修改或删除 | 恢复原始 CSV → 运行 `sync-data.ps1` |
| `datasets.data.js` 为空 | 同步脚本未运行 | 运行 `powershell -File scripts/sync-data.ps1` |
| 产业/文化数据过期 | 超过 24h 未更新 | 运行 `python scripts/daily_data_update.py` |

### 9.5 路径 / 文件问题

| 现象 | 原因 | 解决 |
|------|------|------|
| JSON 解析错误 | 手动编辑破坏了 JSON 格式 | 用 [json.cn](https://www.json.cn/) 粘贴检查 |
| 图片 404 | 路径不一致 | 检查 `industry_ark.json` 中的 `image` 字段与实际文件名匹配 |
| 首页背景白屏 | `images/bg/processed/` 图片缺失 | 运行 `python scripts/process_page_backgrounds.py` |

### 9.6 地图相关问题

| 现象 | 原因 | 解决 |
|------|------|------|
| 地图不显示 | AK 未配置或无效 | 检查 `baiduMapAk` |
| 地图灰色方块 | 百度地图 JS 加载超时 | 检查网络 → 刷新 |
| 点击城市无反应 | 地图缓存过期 | 刷新页面 → 清除 localStorage `yuxiaoqiao_map_*` |
| 地图样式不是深色 | `baiduMapStyleId` 缺失 | 在百度地图控制台创建个性化样式并填入 |

### 9.7 浏览器缓存

| 现象 | 原因 | 解决 |
|------|------|------|
| 改完代码不生效 | 浏览器缓存了旧文件 | `Ctrl + Shift + R` 强制刷新 |
| 数据没更新 | sessionStorage 缓存 | F12 → Application → Clear site data |

## 第十章 项目迁移指南

### 10.1 迁移到另一台电脑

1. 复制整个 `d:\玉小侨demo` 文件夹到新电脑
2. 在新电脑上安装 Python 3.8+（勾选「Add to PATH」）
3. 安装 `requests`：`pip install requests`
4. 运行 `setup-env-check.py` 验证环境
5. 查看 `date/API修改指南.md`，替换其中的 API Key 为新账号的 Key（如果需要）
6. 双击 `打开网页.py` 启动

### 10.2 交接给新开发者

1. 发送整个项目文件夹（或 ZIP 压缩包）
2. 确保包含以下交付物：
   - `开发环境与部署手册.md`（本文档）
   - `date/API修改指南.md`（API 配置指引）
   - `项目手册.md`（技术架构详解）
   - `setup-env-check.py`（环境检查脚本）
3. 告知新开发者：
   - API Key 需要自己在各平台注册获取
   - 不要将 `volc-ark-apis.json` 上传到公开 GitHub

### 10.3 迁移到新的 API 账号

详细步骤参见 **第七章 API 替换指南**。

核心要点：
- 4 个地方要改：`dataApiKey`、`qiaozhuangApiKey`、`baiduMapAk`、`baiduMapStyleId`
- 侨壮壮的 Bot 需要在新账号下重建（Bot ID 绑定账号）
- 全部改在 `date/volc-ark-apis.json` 一个文件即可

### 10.4 迁移到新的数据源

如果后续需要接入真实后端 API（替代 CSV 静态数据）：

1. 后端提供 REST API（返回 JSON）
2. 修改 `js/platform-data-api-config.js`，填入实际 API URL：

   ```js
   window.PLATFORM_DATA_API = {
     cultureUrl: "https://api.example.com/culture",
     industryListUrl: "https://api.example.com/industries",
     // ...
   };
   ```

3. 修改 `js/qiaowu-news-loader.js`，将其从 CSV 读取改为 fetch API
4. 需要后端支持 CORS（跨域）
